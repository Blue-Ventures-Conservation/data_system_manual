Panduan Cepat

Panduan Pendataan Perikanan dan Komunitas

Panduan singkat untuk kerangka kerja dan protokol pengumpulan data survei KoBoToolbox.

Pengantar

Panduan singkat ini memberikan panduan langkah demi langkah untuk mengumpulkan dan memvalidasi data perikanan dan komunitas melalui sistem. Panduan ini merangkum proses pengumpulan data dan menguraikan protokol untuk memperbarui file referensi dan tabel kontrol kualitas.

Kami mengembangkan empat formulir survei untuk mengumpulkan data tentang pendaratan perikanan dan komunitas nelayan. Deskripsi survei terperinci tersedia dalam panduan lengkap dalam terjemahan bahasa Inggris, Bahasa Indonesia, Cebuano, Prancis, Tagalog, Swahili, Malagasi, Spanyol, Portugis, dan Tetun.

Survei Pemantauan Pendaratan menangkap data minimum, idealnya setiap hari atau sesering mungkin, yang diperlukan untuk melacak tren jangka panjang dalam produksi perikanan, komposisi spesies, dan nilai tangkapan. Survei ini dapat membantu mengevaluasi dampak penutupan perikanan atau strategi pengelolaan lainnya dengan membandingkan data sebelum dan sesudah penerapan. Survei ini singkat dan mudah digunakan untuk memungkinkan pengumpulan data yang lebih sering dan konsisten.

Survei Profil Pendaratan mengumpulkan data perikanan terperinci tentang peralatan, upaya, dan ukuran individu ikan atau invertebrata di samping informasi khusus untuk kelompok spesies tertentu seperti jenis kelamin, kualitas tangkapan, data stilet, dan tingkat kematangan gonad. Kualitas tangkapan gurita meliputi cara penyimpanan, warna, jumlah tentakel, lecet fisik, kualitas paruh, dan kondisi kulit. Data stilet gurita meliputi contoh foto dan lingkar pertumbuhan. Data tingkat kematangan gonad memperhitungkan berat gonad, warna, stadium, panjang, dan penampakan garis pada kelenjar ovarium gurita betina. Data ini melengkapi data pemantauan pendaratan dan membantu mitra lebih memahami tren tangkapan yang diamati dalam produksi perikanan dan dampak pengelolaan perikanan. Survei ini dirancang untuk dilakukan pada awal keterlibatan dalam suatu komunitas dan selanjutnya dilakukan setiap satu hingga dua tahun, sebelum dan sesudah menerapkan strategi pengelolaan, atau sesuai kebutuhan untuk menjelaskan perubahan tangkapan yang diamati melalui data pemantauan.

Survei Profil Komunitas mengumpulkan pemahaman yang komprehensif tentang lanskap sosial-ekonomi dan perikanan suatu komunitas. Survei ini digunakan saat pertama kali terlibat dengan komunitas dan diulangi saat diperlukan untuk mendokumentasikan perubahan besar dalam perilaku penangkapan ikan, persepsi nelayan, dan pengelolaan sumber daya. Frekuensi survei akan bergantung pada ruang lingkup proyek (yaitu, kapan perubahan diharapkan terjadi).

Survei Rumah Tangga Nelayan menangkap informasi rumah tangga nelayan yang terperinci, seperti persepsi tingkat rumah tangga tentang kegiatan yang berhubungan dengan penangkapan ikan, sumber pendapatan, kegiatan sehari-hari, spesies ikan utama yang ditangkap, ketergantungan pada ikan untuk makanan, dan kegiatan penangkapan ikan. Survei ini digunakan saat pertama kali terlibat dengan komunitas dan diulangi setiap dua hingga lima tahun atau setelah intervensi pengelolaan untuk mendokumentasikan perubahan perilaku penangkapan ikan, persepsi nelayan, efektivitas pengelolaan, ketahanan pangan, kepercayaan, dan kesetaraan gender.

Langkah-langkah untuk Menggunakan Formulir KoBoToolbox

  1. Siapkan data referensi
    1. Data referensi dapat diupdate melalui Smartsheet dan berisi informasi lengkap tentang tataran administrasi masyarakat nelayan (negara, provinsi, kabupaten / kota, desa/komunitas), lokasi pendaratan, nelayan, pembeli, pengumpul data, data spesies, wilayah pengelolaan, habitat.jenis, jenis roda gigi, dan organisasi mitra.
    2. Lembar data referensi digunakan untuk mengisi daftar tarik-turun dalam formulir survei KoBoToolbox dan menggabungkan data tambahan yang diperlukan untuk analisis seperti koordinat desa dan jenis kelamin nelayan.
    3. Perbarui data referensi dengan spesies baru, nelayan, pembeli, lokasi pendaratan, desa, dll. yang dibutuhkan untuk pendataan di wilayah baru.
    4. Sangat penting bahwa file-file ini mutakhir agar pengumpulan dan analisis data dapat berjalan dengan lancar. File referensi diunggah ke kobo setiap 12-24 jam.
  1. Siapkan Akun KoBoToolbox
    1. Semua survei dikelola melalui penggunaan formulir KoBoToolbox. Setiap formulir survei dapat dibagikan dengan akun KoBoToolbox.
    2. Siapkan Akun Kobo Toolbox Anda di server umum di https://kf.kobotoolbox.org/accounts/signup/
    3. Hubungi tim dengan detail akun Kobo dan nama pengguna Anda agar formulir yang sesuai dapat dibagikan dengan akun Anda.

  1. Unduh Aplikasi KoboCollect.

    1. Formulir Kobo dapat diakses secara gratis melalui aplikasi ponsel Android KoboCollect atau melalui formulir web Enketo.

  2. Muat formulir di Aplikasi KoboCollect. Lihat Pengaturan Kobo untuk panduan tambahan.

    1. Setelah aplikasi diunduh, siapkan URL server, Nama Pengguna, dan Kata Sandi, yang menghubungkan aplikasi KoboCollect Anda ke server KoBoToolbox. Ini memungkinkan Anda untuk mengunduh formulir yang telah dibagikan dari KoBoToolbox ke perangkat seluler Anda dan juga mengirim data yang dikumpulkan melalui aplikasi kembali ke server. Untuk URL, gunakan URL server penggunaan umum - https://kc.kobotoolbox.org.
    2. Anda dapat mengakses “Pengaturan Proyek” dengan mengklik ikon di sudut kanan atas layar beranda.
    3. Di layar beranda, pilih ” Unduh Formulir.”
    4. Daftar semua formulir survei yang Anda gunakan akan muncul. Tekan “Pilih Semua” agar semua formulir survei dikirim ke aplikasi atau pilih formulir yang ingin Anda miliki dengan memilihnya secara manual. Kemudian, klik “Terpilih.”
    5. Penting agar survei diperbarui secara berkala untuk memastikan versi terbaru formulir digunakan agar tidak menimbulkan masalah dalam alur data. Meskipun formulir web diperbarui secara otomatis saat versi baru survei tersedia, pengguna harus mengonfigurasi “pembaruan otomatis” di aplikasi. Anda dapat mengatur pembaruan otomatis dengan membuka “Pengaturan”dan”Manajemen formulir”.
  1. Kumpulkan data dengan formulir survei KoBoToolbox
    1. Kumpulkan data dengan memilih “Mulai Formulir Baru” di aplikasi KoboCollect. Lihat petunjuk terperinci tentang pengumpulan data menggunakan aplikasi di Pengaturan Kobo.
    2. Di akhir survei, pilih “Finalisasi.”Jika Anda ingin mengedit formulir sebelum mengirimkan, pilih” Simpan sebagai draf.”Jika Anda memilih ” Simpan sebagai draf”, formulir Anda akan dikirim ke ” Draf”. Di sini, Anda dapat mengedit formulir sebelum mengirimkannya ke kumpulan data akhir. Setelah Anda membuat perubahan, pilih “Finalisasi”, seperti yang disebutkan di atas. Setelah memilih “Finalisasi”, formulir akan disimpan secara otomatis di “Siap dikirim”. Dalam versi Kobo saat ini, Anda masih dapat mengedit formulir di sini. Namun, di masa mendatang, Anda tidak akan dapat mengeditnya setelah difinalisasi, dan “Siap dikirim” hanya akan menjadi tempat penyimpanan untuk menyimpan formulir yang telah difinalisasi hingga Anda memiliki koneksi internet untuk mengirimkannya ke server. Pilih “Kirim yang dipilih” untuk mengirim semua formulir ke server.
    3. Data yang dikumpulkan menggunakan aplikasi atau formulir web akan dibersihkan dan disimpan dalam sistem penyimpanan data pusat. Data mentah dicadangkan dalam basis data KoBoToolbox.
  1. Validasi data dalam Smartsheet
    1. Semua data yang mengalir melalui sistem data perikanan divalidasi dalam Smartsheet. Panduan instruksi langkah demi langkah lengkap untuk menyelesaikan proses ini dapat ditemukan di halaman Validasi Data.
    2. Data dikirim untuk validasi dalam dua skenario; 1) nama baru untuk nelayan, pembeli, pengumpul data, atau spesies ditambahkan secara manual di Kobo Collect, atau 2) nilai numerik melampaui ambang batas yang ditetapkan. Entri ini harus diverifikasi sebelum disertakan dalam kumpulan data akhir, divisualisasikan di dasbor, dan ditambahkan ke file referensi.
    3. Setiap nama baru atau data yang ditandai dikirim ke tabel quality control yang relevan di Smartsheet untuk divalidasi oleh tim lapangan. Tabel quality control bersifat mitra spesifik dan dapat diakses melalui permintaan pembaruan yang dikirim ke email Anda atau dengan mengunjungi tautan Tampilan Dinamis yang disediakan untuk Anda. Pimpinan tim lapangan akan diberi tahu melalui email saat data perlu divalidasi dalam tabel.
    4. Memvalidasi nama baru: Saat nama baru dimasukkan selama pengumpulan data, entri ini secara otomatis direferensikan silang dengan file referensi untuk menilai apakah entri baru mungkin sudah ada di file referensi, tetapi mungkin dieja secara berbeda. Entri serupa yang ada akan ditambahkan ke tabel di bawah kolom “ref_match”. Pemimpin tim kemudian dapat dengan mudah memverifikasi apakah data “baru” sebenarnya sudah ada di file referensi. Pemimpin tim akan melengkapi kolom ” validation_options” di setiap tabel dengan memilih new_data, entri data referensi silang dari file referensi (yaitu “ref_match”), other, atau remove. Jika “other” dipilih, Anda harus memberikan nama di kolom “other”. Jika itu adalah nama spesies lokal baru yang perlu divalidasi, kolom dengan foto yang diambil selama pengumpulan data akan disediakan untuk referensi.
    5. Memvalidasi nilai numerik dengan peringatan ambang batas: Untuk entri data yang ditandai, pemimpin tim akan menyelesaikan validasi data di kolom “correct” dengan memilih Y (ya, nilai sudah benar), N (tidak, nilai tidak benar), atau hapus (data tidak dapat divalidasi dan harus dihapus dari master kumpulan data). Jika memilih “tidak”, nilai yang benar untuk pembilang atau penyebutnya harus dimasukkan di kolom “correct_numerator” dan “correct_denominator”.
    6. Setelah data diverifikasi dalam tabel validasi, kami akan menambahkan informasi baru ke file referensi dan memberikan koreksi yang diperlukan pada master kumpulan data.

  1. Perbarui lembar terjemahan

    1. Terjemahan untuk semua bahasa diselesaikan dengan mengedit lembar yang sesuai di Smartsheet.

  2. Visualisasikan dan gunakan data

    1. Analisis dan visualisasi otomatis akan tersedia melalui dasbor terstandarisasi. Kumpulan data juga akan tersedia melalui unduhan di dasbor atau tautan langsung ke basis data lain.

Beri tahu kami jika Anda memiliki pertanyaan.

Terima kasih!
